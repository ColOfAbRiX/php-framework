buffer_add_handler  [line 216]

void buffer_add_handler( String $handler, mixed $handler_file, [Integer $level = -1]  )

Aggiunge un gestore di buffer

Aggiunge un gestore della terminazione di buffering del livello specificato. Tra tutti i gestori assegnati ad un livello di buffer, l'ultimo inserito ha la priorità maggiore. Se $level è inferiore a 1 viene impostato a 1, se superiore all'ultimo livello amissibile viene considerato pari a questo.

 Parameters:
String   $handler:  Il nome della funzione che gestisce il livello di buffering
Integer   $level:  Il livello a cui assegnare l'handler

API Tags:
Global:  Array $buffer_handlers: L'insieme di tutti i gestori degli errori per i livelli di buffering

String buffer_call_handlers( )

Chiama i gestori del livello di buffer più elevato

La fuzione si occupa di richiamatre tutti i gestori dell'ultimo livello di buffer specificato, chiamando per primo il gestore inserito per ultimo. Se siamo al primo livello, nel quale non è più possibile gestire il codice, viene creato un buffer temporaneo.
API Tags:
Return:  L'output di tutti i gestori chiamati per l'ultimo livello

void buffer_remove_handler( Integer $level, String $handler, [Bool $remove_all = false]  )

Rimuove un gestore di buffer.

Rimuove un gestore del livello di buffer dall'insieme dei gestori per quel livello, oppure li elimina tutti. Se $level è inferiore a 1 viene impostato a 1, se superiore all'ultimo livello amissibile viene considerato pari a questo.

 Parameters:
Integer   $level:  Il nome della funzione che gestisce il livello di buffering
String   $handler:  Il livello a cui eliminare l'hander
Bool   $remove_all:  Se TRUE elimina tutti i gestori del livello e non considera $handler

API Tags:
